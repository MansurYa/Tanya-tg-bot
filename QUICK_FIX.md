# Краткая инструкция для сервера

## Текущая ситуация
На сервере запущено несколько экземпляров бота → конфликт Telegram API

## Решение (выполнить на сервере)

```bash
# 1. Перейти в директорию проекта
cd /root/Tanya-tg-bot  # или ваш путь

# 2. Подтянуть изменения
git pull origin main

# 3. Сделать скрипты исполняемыми
chmod +x *.sh

# 4. Остановить все экземпляры бота
./stop_bot.sh

# 5. Запустить один экземпляр
./start_bot.sh

# 6. Проверить статус
./check_bot.sh

# 7. Посмотреть логи (опционально)
tail -f bot.log
```

## Что будет исправлено
✅ Конфликт множественных экземпляров
✅ Дублирующиеся уведомления (старая + новая система)
✅ Правильные сообщения из датасета

## Проверка работы
После запуска пользователи будут получать сообщения в формате:
```
Для тебя 💌

[комплимент из датасета]

"[психологический принцип]"
```

## Если что-то пошло не так
См. подробные инструкции:
- FIX_CONFLICT.md - решение конфликта экземпляров
- SERVER_FIX_INSTRUCTIONS.md - полная инструкция по деплою
