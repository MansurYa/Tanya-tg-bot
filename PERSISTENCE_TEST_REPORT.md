### 3. Планирование сообщений

**Механизм:**
- **Ежедневно в 00:05** - функция `schedule_morning_messages()` планирует сообщения для всех пользователей
- **Для каждого пользователя** генерируется случайное время: `08:45 ± 67 минут` (нормальное распределение)
- **APScheduler** создаёт задачу с уникальным ID: `morning_msg_{user_id}_{date}`

**Результат:** ✅ РАБОТАЕТ

```python
# Генерация времени
mean_time = datetime.combine(today, time(9, 45))
std_minutes = 67
random_minutes = int(np.random.normal(0, std_minutes))
send_time = mean_time + timedelta(minutes=random_minutes)

# Планирование
scheduler.add_job(
    send_morning_message,
    'date',
    run_date=send_time,
    args=[bot, user_id],
    id=f'morning_msg_{user_id}_{today}',
    replace_existing=True
)
```

async def restore_state(update: Update, context: CallbackContext) -> int:
    user_id = update.effective_user.id
    user = get_user(user_id)

    if not user:
        return await start(update, context)

    if user["state"] == "COMPLETED":
        await show_main_menu(update, context)
        return COMPLETED

    # Восстанавливает текущий вопрос
    q_idx = user.get("current_question", 1)
    if q_idx == 1:
        return await handle_answer_q1(update, context)
    elif q_idx == 2:
        return await handle_answer_q2(update, context)

## ⚠️ Обнаруженная проблема

### Проблема: Пропуск сообщений при перезапуске после 00:05

**Сценарий:**
1. Бот запланировал сообщения в 00:05
2. Бот перезапустился в 10:00
3. Запланированные задачи потеряны (APScheduler не персистентный)
4. Пользователи не получат сообщения сегодня

**Решение:** ✅ ИСПРАВЛЕНО

Добавлен вызов `schedule_morning_messages()` при старте бота:

```python
async def setup_scheduler(app: Application):
    # ... настройка cron задач ...

    scheduler.start()
    logger.info("Планировщик запущен")

    # ✅ Перепланирование отложенных сообщений
    await schedule_morning_messages(app.bot)
    logger.info("Проверка и планирование отложенных сообщений завершена")
```

**Теперь:**
- При старте бота проверяются все пользователи
- Если пользователь должен получить сообщение сегодня, но ещё не получил - планируется заново
- Время генерируется заново (случайное в пределах окна)

## 🎯 Гарантии системы

1. **Каждый пользователь, завершивший квест, получит утреннее сообщение**
   - Планируется ежедневно в 00:05
   - Перепланируется при перезапуске бота
   - Отправляется в случайное время 08:45 ± 67 минут

2. **Сообщения не повторяются внутри цикла**
   - 73 комплимента показываются по одному разу
   - 125 цитат показываются по одному разу
   - После исчерпания - автоматический сброс

3. **Состояние сохраняется между сессиями**
   - Прогресс квеста
   - Показанные сообщения
   - Настройки пользователя

4. **Сообщения отправляются в правильные чаты**
   - Каждый пользователь имеет уникальный telegram_id
   - Бот использует этот ID для отправки
