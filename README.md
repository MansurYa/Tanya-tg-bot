# Tanya Telegram Bot

Персонализированный Telegram бот с системой квестов и утренними сообщениями.

## Возможности

- **Система квестов**: Интерактивный квест с 12 подарками и вопросами
- **Утренние сообщения**: Ежедневные автоматические сообщения с комплиментами и психологической поддержкой
- **Поддержка**: Эмоциональная поддержка и виртуальные объятия по запросу
- **Ротация сообщений**: Умная система ротации без повторов внутри цикла

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Настройте бота:
```bash
cp config.example.json config.json
# Отредактируйте config.json, добавив токены и API ключи
```

3. Запустите миграцию базы данных:
```bash
python migrate_db.py
```

4. Запустите бота:
```bash
# Локально (для разработки)
python -m src.bot

# На сервере (в фоне)
nohup python -m src.bot > bot.log 2>&1 &

# Или используйте скрипт
./start_bot.sh
```

## Проверка работы

После запуска проверьте:
```bash
# Статус бота
./check_bot.sh

# Загрузку датасетов
python verify_datasets.py

# Логи
tail -f bot.log
```

## Структура проекта

```
/src
  ├── bot.py              # Основной модуль бота
  ├── database.py         # Работа с SQLite
  └── message_pool.py     # Логика ротации сообщений

/tanya_dataset_generator
  ├── output_v2/          # Финальные датасеты
  ├── stages_v2/          # Скрипты генерации датасетов
  └── utils_v2/           # Утилиты для генерации

config.json              # Конфигурация бота (не в git)
data.db                  # База данных SQLite (не в git)
```

## Управление ботом на сервере

### Проверка статусов
```bash
./check_bot.sh           # Проверить статус бота
python verify_datasets.py # Проверить загрузку датасетов
```

### Управление процессами
```bash
./stop_bot.sh            # Остановить все экземпляры
./start_bot.sh           # Запустить бота
./deploy_fix.sh          # Автоматический деплой с исправлениями
```

### Исправление базы данных
```bash
python fix_duplicate_notifications.py  # Отключить старую систему уведомлений
```

## Безопасность

- Никогда не коммитьте `config.json` с реальными токенами
- Используйте `config.example.json` как шаблон
- Храните `data.db` только локально
- Бэкапы базы данных создаются автоматически при запуске `fix_duplicate_notifications.py`
