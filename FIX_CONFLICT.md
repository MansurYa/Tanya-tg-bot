# Исправление конфликта "multiple bot instances"

## Проблема
```
telegram.error.Conflict: terminated by other getUpdates request;
make sure that only one bot instance is running
```

Это означает, что на сервере запущено несколько экземпляров бота одновременно, и они конфликтуют при получении обновлений от Telegram API.

## Быстрое решение

```bash
cd /path/to/Tanya-tg-bot
git pull origin main
chmod +x *.sh

# Остановить все экземпляры
./stop_bot.sh

# Запустить один экземпляр
./start_bot.sh

# Проверить статус
./check_bot.sh
```

## Подробные команды

### Проверить статус бота
```bash
./check_bot.sh
```

Покажет:
- ✅ Один экземпляр работает (нормально)
- ⚠️ Несколько экземпляров (конфликт!)
- ❌ Не запущен

### Остановить все экземпляры
```bash
./stop_bot.sh
```

Безопасно остановит все запущенные экземпляры бота.

### Запустить бота
```bash
./start_bot.sh
```

Запустит бота в фоне и проверит, что он успешно стартовал.

## Почему это происходит?

1. Бот был запущен несколько раз без остановки предыдущего
2. Скрипт деплоя не убил все процессы
3. Есть "зомби" процессы

## Как избежать в будущем?

Всегда используйте скрипты:
```bash
# Перезапуск
./stop_bot.sh && ./start_bot.sh

# Или автоматический деплой
./deploy_fix.sh
```

## Ручная проверка (если скрипты не работают)

```bash
# Найти все процессы
ps aux | grep "python -m src.bot"

# Убить все
pkill -9 -f "python -m src.bot"

# Проверить, что убиты
ps aux | grep "python -m src.bot"

# Запустить заново
nohup python -m src.bot > bot.log 2>&1 &

# Проверить логи
tail -f bot.log
```
